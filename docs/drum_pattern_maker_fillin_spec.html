1. 目的と前提

この仕様書の目的は、Drum Pattern Maker にフィルイン学習機能を実装するにあたり、(a) 既存教材の取り込み方、(b) 内部データモデル、(c) 楽譜表示ルール、(d) オーディオ再生ルール、(e) 将来拡張可能な分類体系、をひとつの基準にまとめることです。

最低限やるべきこと

  • DARYL’s Drum Channel Lesson 15 の 100 フィルインを基礎コーパスとして管理する。
  • フィルインを 1拍 / 2拍 / 3拍 / 4拍 の句長で管理する。
  • 8分・16分・8分三連・16分三連・混合 のどのグリッドでも内部表現できるようにする。
  • 表示（abcjs等）と再生（Tone.js等）は必ず 同じ正規化済みイベント列 から作る。

この文書の運用方針

  • 原典画像は下部の「原典画像アーカイブ」に全件掲載。
  • 原典画像は 外部参照URL で埋め込み。長期運用時はローカルミラー化推奨。
  • 楽譜表示は abcjs、音は Tone.js を前提としたハイブリッドを基本案とする。
  • 権利処理が不明な外部画像は 社内参照用 として扱い、公開配布時は自社生成譜面へ置換する。
重要: 原典ページは 1拍・2拍・3拍・4拍のフレーズを含む 100 種類のフィルインを楽譜付きで公開しており、最初に楽譜記号の読み方の画像を掲示しています。文書後半では、この原典を土台にしつつ、追加で triplet / linear / rudiment-derived fill まで包含する仕様へ拡張しています。

4. 譜面仕様（内部表現と表示仕様）

4.1 記譜原則

内部仕様は PAS 系ガイドラインの考え方に寄せる。つまり、伝統的記譜法の延長で、見た瞬間に認識しやすく、記号の意味が一意で、表記キーを必ず先頭に出す ことを原則とする。特にドラムセットは「絶対標準」がないため、毎回 legend / key を先頭に出し、同一出版物・同一画面では記号の意味を固定する。

項目採用仕様備考
五線5線 + neutral/percussion clefドラムセット全体は音高よりも位置識別を優先
キー（legend）全作品の先頭に必須使用する全楽器、note head、articulation、beater、sticking を定義
スネア第三間（canonical internal rule）調査文献でも最多派
バスドラム第一間（canonical internal rule）調査文献でも多数派
ハイハット（手）譜表上方のスペース、x-notehead日本式運用では closed HH = x-notehead、open HH = x-notehead の上に open circle（◯）を付与して区別
ハイハット（足）譜表下のスペースfoot splash / foot close は別 articulation。手のハイハット open/close 記号とは分離して管理
ライド上線x-notehead
クラッシュ譜表上の補助線x-notehead
タム内部 ID で管理し、表示時に kit layout に応じて割当2タム/3タム/4タム構成差に強くする

4.2 内部時間表現

  • meter: デフォルト 4/4
  • phrase_length_beats: 1 / 2 / 3 / 4
  • subdivision: eighth / sixteenth / eighth_triplet / sixteenth_triplet / mixed
  • canonical grid: 1拍 = 12 tick を基準にして、8分・16分・三連を共通変換可能にする（例: 1拍12tickなら 8分=6tick, 16分=3tick, 8分三連=4tick）。
  • 表示は abcjs / SVG / Canvas に合わせて変換、再生は Tone.js イベント列に変換。

4.3 表示用の canonical event schema

type DrumEvent = {
  tick: number;              // 小節頭からの絶対 tick
  durationTick: number;      // 音価
  voice: "snare" | "kick" | "hh" | "ride" | "crash" | "splash" | "hiTom" | "midTom" | "lowTom" | "rim";
  hand?: "R" | "L" | "K";
  dynamic?: "ghost" | "weak" | "normal" | "accent" | "max";
  articulation?: "closed" | "open" | "flam" | "drag" | "buzz" | "rimshot" | "crossStick" | "choke";
  accent?: boolean;
  grace?: boolean;
  source?: "daryl15" | "drumeo" | "manual" | "generated";
};

4.4 表示ルール

  • 初心者向けモードでは、音価ごとのグリッド、強弱、句長の把握を優先し、必要なら簡略表示を許可する。
  • 上級モードでは、ゴーストノート、フラム、ドラッグ、オープン/クローズHH まで表現する。
  • ハイハット表記の日本式補正: closed HH は通常の x-notehead。open HH は x-notehead の上に open circle（◯）を追加して表示する。先行版仕様書で open / close の対応が逆転していたため、本版で修正した。
  • 原典ページの「6 HihatOpen」相当画像に描かれている通常の x-notehead は、本仕様では closed HH として扱う。open HH は必ず「x + 上部◯」で正規化する。
  • キックとハンド系イベントは同 tick に共存可能。
  • landing beat（多くは次小節頭）を強調表示できると学習効率が高い。

4.5 ハイハット open / close 正規化ルール（日本式）

対象表示内部ラベル実装メモ
Closed Hi-Hatx-notehead のみinstrument: "hihat", articulation: "closed"通常の手ハイハット。先行仕様書で open と誤対応していた箇所を修正。
Open Hi-Hatx-notehead + 上部 open circle（◯）instrument: "hihat", articulation: "open"表示レイヤーでは x 記号と open circle を別パーツで重ねる。再生エンジンでは sustain を持たせ、後続の closed / foot close で choke 可能にする。
Foot Close足ハイハット用の独立記号instrument: "hihatFoot", articulation: "close"手の closed HH と混同しない。open HH の停止トリガーとしても利用可。
 
{
  "instrument": "hihat",
  "articulation": "open",
  "display": {
    "notehead": "x",
    "openCircle": true
  },
  "playback": {
    "sample": "hh_open",
    "sustain": true,
    "canBeChokedBy": ["hihat.closed", "hihatFoot.close"]
  }
}

6. 生成・再生エンジン仕様

6.1 生成エンジン

  • 表示レイヤは abcjs 等に任せても、生成の真実は必ず canonical event sequence に置く。
  • 原典画像しかないパターンは、最初に semi-structured annotation を作る。
  • annotation 手順: 画像参照 → 句長推定 → subdivision 推定 → voice sequence 入力 → sticking/dynamic 補完 → human QA。
  • 最終的には「source image を見せなくても」内部イベントだけで譜面再構成できる状態にする。

6.2 Tone.js 再生ルール

  • kick = 独立音色、snare weak = 弱いノイズ系 / short transient、snare accent = 同音色の gain 増し、tom = pitch-mapped membrane、crash = metallic noise / sample。
  • ユーザーが再生を押した後に audio context を開始する。
  • 再生カーソルは表示譜面と同じ step index を使って光らせる。
  • 将来的に 1行再生 / 全行連続再生 / ループ再生 / カウントイン再生 を選択可能にする。

6.3 abcjs 表示レイヤ

  • abcjs は表示用。原則として「学習データの正規化」と「音声生成」は abcjs に依存しない。
  • abcjs に渡す前に internal event → notation token 変換レイヤを設ける。
  • 今後、初心者用簡略譜 / 標準譜 / 練習用分解表示 の 3モードを切替可能にする。

7. 学習カリキュラムと難易度定義

レベル到達目標解禁カテゴリ表示推奨
Lv.1句長1拍、8分/16分の単純フィルを拍頭に着地straight_8th / straight_16th簡略譜 + 音価グリッド
Lv.22拍〜4拍のストレート fill、タム回し、弱アクセントbuild / around_the_kit / kick_assisted簡略譜 + 強弱
Lv.3Motown / mnemonic / phrase ending を理解motown / motif_fill標準譜 + カウント表示
Lv.48分三連 / 16分三連 / kick displacementtriplet_8th / triplet_16th標準譜 + subdivision 補助
Lv.5linear / rudimental / Bonham triplet を音楽的に使うlinear / rudimental / bonham標準譜 + sticking + dynamics

推奨順序

  1. 1拍フィル → 2拍フィル → 4拍フィル
  2. 8分 → 16分 → 8分三連 → 16分三連
  3. スネア中心 → タム回し → キック混合 → linear
  4. single stroke → accent pattern → rudiment derived
