# VexFlowドラム譜面のチューニング計画

## 課題と原因分析
1. **右側の見切れ**
   - 原因: 画面幅より広い楽譜を描画しようとした際に、親コンテナがスクロールを受け付けずSVGがはみ出していたため。
2. **ハイハットとスネア等の縦ズレ・不自然な符尾（Stem）**
   - 原因: VexFlowでは、同じ拍に「上向き（Stem Up）」の独立した声部（Voice）が複数存在すると、衝突を避けるために横にずらして描画する仕様になっています。
   - 対策: **ドラムの「手（ハイハット・シンバル・スネア・タム）」をすべて声部1（上向き）にまとめ、和音（Chord）扱いにすることで完全に縦を揃えます。**「足（キック）」は声部2（下向き）に分離します。
3. **16分音符がバラバラになる / 休符が異常に多い**
   - 原因: グルーピング処理（8分や4分への間引き）のロジックが不均一でした。
   - 対策: `16th`, `8th`, `4th` のグリッドに応じて適切に音符の長さを結合する処理に統一し、VexFlow標準のBeat（拍）単位で連桁（Beam）を引きます。

## 変更内容：`src/components/VexFlowNotationPreview.jsx`

### 1. 声部（Voice）の統合と完全分離
- **Voice 1 (Stem Up):** 手で叩く要素（ハイハット `g/5/x2`、シンバル `g/5/cx`、スネア `c/5`、タム `e/5`、フロアタム `a/4`）をすべてここに統合します。
- **Voice 2 (Stem Down):** 足で踏む要素（キック `f/4`）のみを独立させます。
- 同時に鳴る場合は、`['g/5/x2', 'c/5']` のように1つの和音としてVexFlowに渡すことで、横ズレ（オフセット）を完全に防ぎます。

### 2. 休符・持続音のスマートコンプレス（圧縮）
- リズムの最小単位（modeや難易度による）を判定し、「空のステップ」が続く場合は自動で8分音符や4分音符の長さにまとめて描画します。
- 鳴らない部分は見えない休符（GhostNote）にすることで、ゴミのような休符記号が散らばるのを防ぎます。

### 3. ビーム（連桁）の標準化
- ハイハットを個別で切り離すといった特殊な処理は行わず、**「1拍（Quarter Note）ごとにビームでまとめる」**という標準的な楽器譜の仕様を適用します。これにより、16分フィルインも綺麗に4つずつ繋がります。

### 4. レスポンシブ・スクロール対応
- コンテナの `div` に `overflowX: 'auto'` を付与し、スマホ等で小節数が多い場合は右にスクロールしてすべて見られるようにします。
- 1小節あたりの最低ピクセル数（約240px）を担保し、音符が重なりすぎるのを防ぎます。

## 実装時の影響
- スネア・タム・ハイハットなど「手」で叩く音符の棒（ステム）はすべて**上向き**に統一され、キックの棒は**下向き**になります。
- 同時に叩く音（例：ハイハット＋スネア）は1本の縦棒（ステム）を共有する形になり、ズレが完全に解消されます。
- 楽譜が横に長い場合は、画面右端で途切れるのではなく「横スクロール」で最後まで確認できるようになります。
